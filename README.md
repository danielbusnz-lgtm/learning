# Learning

Personal study repo. Reading material, language practice, interview prep, and progress notes — kept separate from runnable projects (those live in `~/Projects`).

## Layout

```
books/              Reading material (PDFs + chapter splits) by domain
  algorithms/         Grokking Algorithms
  business/           accounting, startup
  c/                  K&R
  data-analysis/      Python for Data Analysis
  editor/             Practical Vim
  hacking/            Hacking: The Art of Exploitation
  linux/              Linux Command Line & Shell Scripting Bible
  math/               Calculus, Discrete, Linear Algebra
  ml/                 Hands-On ML (Géron) + Deep Learning Foundations (Bishop)
  networking/         Computer Networking: A Top-Down Approach
  python/             Fluent Python
  rust/               The Rust Programming Language
  software/           Clean Code, Database Internals, Software Architecture, System Design Interview, Pragmatic Programmer
  stats/              Statistics (Freedman/Pisani/Purves)
  writing/            On Writing Well

languages/          Hands-on practice code by language
  go/
  javascript/
  python/             practice scripts + pytorch project
  sql/

interview-prep/     DSA + system design practice
  leetcode/

notes/              Cross-cutting tracker files
  progress.md
  best-books.md
  oreilly-books.md
```

## Conventions

- All folders are `kebab-case`.
- Books that have been split into chapter PDFs have both the full PDF and a `*-chapters/` or per-chapter file layout in the same domain folder.
- Runnable projects (with their own git repos and deployments) live in `~/Projects`, not here. Examples: `~/Projects/HiveNet`, `~/Projects/california-housing-ml`.
